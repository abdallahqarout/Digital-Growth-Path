What is OOP?

📘
Core Overview Fundamentals
Object-Oriented Programming (OOP) is a programming paradigm that organizes code into classes and objects. Core principles include encapsulation, which protects data within objects; inheritance, which allows classes to share properties and behaviors; polymorphism, which enables objects to be used interchangeably; and abstraction, which hides unnecessary details. OOP improves code modularity, readability, and reusability, making it easier to manage complex software. Languages like Java, C++, and Python use OOP to build applications ranging from desktop programs to large-scale enterprise systems.
🌍
Real-World Model Industry Use
Object-Oriented Programming models real-world entities as objects. Each object contains attributes (data) and methods (functions) that define its behavior. Classes act as blueprints for creating objects, ensuring consistency and structure. OOP encourages modularity, allowing developers to divide complex systems into manageable pieces. Inheritance lets a class adopt features from another, reducing code duplication and enhancing reuse. Polymorphism allows objects of different classes to be treated uniformly, while abstraction hides unnecessary details — exposing only relevant functionality to the user. OOP is widely used in software design, enabling maintainable, scalable, and extensible applications across languages like Java, C++, Python, and C#.

The 4 Pillars of OOP

🔒
Encapsulation Data Protection
Bundles data and the methods that operate on it into a single object. Restricts direct access to internal state, exposing only what is necessary through public methods. Prevents accidental corruption of data and keeps internal logic hidden.

Example: A bank account hides the balance — you access it only via deposit() or withdraw().

🧬
Inheritance Code Reuse
Allows a child class to inherit properties and methods from a parent class, avoiding duplication and enabling a shared foundation. Child classes can extend or override inherited behavior to add specialization.

Example: A Dog class inherits from Animal — it gets eat() and sleep() but adds its own bark().

🔀
Polymorphism Flexibility
Enables objects of different classes to be treated as instances of the same parent class. The same method call can produce different results depending on the actual object type — making code more flexible and extensible.

Example: Both Cat and Dog have speak() — Cat returns "Meow", Dog returns "Woof".

🎭
Abstraction Simplification
Hides the complex internal implementation and exposes only the essential features of an object. Users interact with a simplified interface without needing to understand the underlying logic.

Example: You drive a car with a steering wheel — you don't need to know how the engine works.

Class vs. Object

A Class is the blueprint — Objects are the instances created from it:
Class (Blueprint)
class Car {
  brand
  speed
  drive()
  stop()
}
→
Object 1
brand = "Toyota"
speed = 120
Object 2
brand = "BMW"
speed = 200
